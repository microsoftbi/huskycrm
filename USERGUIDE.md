## 1. 系统概述

Husky CRM 是一套轻量级客户关系管理系统，帮助销售团队高效管理客户、联系人、销售机会和日常拜访活动。

### 核心能力

| 功能 | 说明 |
|------|------|
| **账户管理** | 管理客户/公司信息，支持搜索、分页、筛选 |
| **联系人管理** | 管理客户方联系人，支持关联到账户 |
| **产品管理** | 维护产品目录，含分类和定价 |
| **销售机会** | 跟踪销售管道，Kanban 看板视图，支持阶段拖拽 |
| **拜访管理** | 计划与执行客户拜访，签到/签退，任务清单 |
| **销售区域** | 层级化区域管理，分配成员和账户 |
| **自定义对象** | 按需创建数据表和字段，扩展业务数据管理 |
| **工作流** | 自动执行规则，满足条件时触发通知等动作 |
| **报表与仪表盘** | 数据分析和可视化展示 |

### 系统要求

- 浏览器：Chrome 90+ / Firefox 90+ / Edge 90+ / Safari 15+
- 推荐分辨率：1366×768 以上
- 无需安装任何客户端软件

## 4. 账户管理

账户是 CRM 的核心，代表一个客户公司或组织。

### 4.1 账户列表

**入口**：侧边栏 → 账户

**功能**：
- 浏览所有账户的表格视图
- 按名称搜索（搜索框输入关键词）
- 分页浏览
- 点击行进入详情页

### 4.2 新建账户

1. 进入账户列表页
2. 点击"新建账户"按钮
3. 填写表单：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 账户名称 | ✅ | 公司/组织全称 |
| 行业 | ❌ | 所属行业分类 |
| 电话 | ❌ | 联系电话 |
| 网站 | ❌ | 公司网址 |
| 邮箱 | ❌ | 联系邮箱 |
| 账单地址 | ❌ | 街道、城市、省份、邮编、国家 |
| 描述 | ❌ | 备注信息 |

4. 点击"保存" → 跳转到新账户详情页

### 4.3 编辑账户

1. 进入账户详情页
2. 点击"编辑"按钮
3. 修改需要更新的字段
4. 点击"保存"

### 4.4 删除账户

1. 进入账户详情页
2. 点击"删除"按钮
3. 在确认弹窗中点击"删除"
4. 跳转回列表页

> ⚠️ 删除操作不可撤销。删除账户后，关联的联系人将不再关联该账户。

3. 填写表单：

| 字段 | 必填 | 说明 |

| 姓 | ✅ | 联系人姓氏 |
| 名 | ✅ | 联系人名字 |

| 手机 | ❌ | 移动电话 |

4. 点击"保存"

> 💡 **小技巧**：新建联系人时选择"所属账户"，后续在账户详情页的"联系人" Tab 中可以直接查看该账户下的所有联系人。

## 7. 销售机会与管道看板

销售机会（Opportunity）追踪从初步接触到最终成交的完整销售过程。

### 7.1 销售阶段

系统预设 7 个标准阶段：

| 阶段 | 成功率 | 说明 |
|------|:------:|------|
| 初步接触 | 10% | 首次接触客户 |
| 需求分析 | 30% | 了解客户需求 |
| 方案制定 | 50% | 提供解决方案 |
| 商务谈判 | 70% | 价格和条款谈判 |
| 合同签订 | 90% | 准备合同 |
| 赢单 | 100% | 成功成交（关闭） |
| 输单 | 0% | 丢失机会（关闭） |

### 7.2 新建商机

1. **入口**：侧边栏 → 销售机会 → 新建商机
2. 填写表单：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 机会名称 | ✅ | 商机标题 |
| 关联账户 | ✅ | 对应的客户 |
| 销售阶段 | ✅ | 当前所处阶段 |
| 金额 | ❌ | 预估金额 |
| 成功率 | ❌ | 根据阶段自动填充，可手动调整 |
| 预计关闭日期 | ❌ | 预期的成交日期 |
| 描述 | ❌ | 详细说明 |

3. 点击"保存"

### 7.3 管道看板

**入口**：侧边栏 → 销售机会 → 管道看板

管道看板以 Kanban 风格展示所有商机，按阶段分组：

## 8. 拜访管理

拜访管理（Event/Task）覆盖客户拜访的完整生命周期：计划 → 准备 → 签到 → 执行 → 签退 → 跟进。

### 8.1 拜访列表

**入口**：侧边栏 → 拜访

**功能**：
- 浏览所有拜访事件
- 按主题搜索
- 按状态筛选（全部/计划中/进行中/已完成/已取消）
- 按类型筛选（拜访/电话/视频会议）
- 点击行进入详情页

### 8.2 新建拜访

1. 点击"新建拜访"
2. 填写表单：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 拜访主题 | ✅ | 简要描述拜访目的 |
| 拜访类型 | ❌ | Visit / Phone Call / Video Conference / Client Visit / Other |
| 关联账户 | ❌ | 关联的客户（WhatId） |
| 关联商机 | ❌ | 关联的销售机会 |
| 联系人 | ❌ | 拜访对象（WhoId） |
| 计划开始时间 | ✅ | 计划拜访时间 |
| 计划结束时间 | ❌ | 预计结束时间 |
| 拜访目的 | ❌ | 详细说明拜访目的 |
| 拜访准备 | ❌ | 提前准备的材料和要点 |
| 位置 | ❌ | 拜访地点 |

3. 点击"保存"

### 8.3 拜访执行流程

拜访执行分为三个步骤：

#### 步骤一：签到

在拜访详情页点击"签到"按钮：
- 系统记录实际开始时间（`actual_start_time`）
- 状态自动变为"进行中"
- 可输入签到位置信息

#### 步骤二：填写拜访记录

签到后填写：
- **拜访纪要**：会议内容和讨论要点
- **拜访结果**：success / neutral / failure / no_show
- **下一步计划**：后续跟进事项

#### 步骤三：签退

填写完毕后点击"签退"按钮：
- 系统记录实际结束时间（`actual_end_time`）
- 自动计算拜访时长
- 状态自动变为"已完成"

### 8.4 任务清单

每个拜访事件可以关联多个 Task（任务）：

| 场景 | 任务示例 |
|------|---------|
| **会前准备** | 准备产品样机、打印技术文档、预定会议室 |
| **会后跟进** | 发送会议纪要、准备报价单、安排下次拜访 |

**操作**：
1. 进入拜访详情页 → "任务清单" Tab
2. 点击"添加任务" → 输入任务主题
3. 完成任务后点击复选框标记为"已完成"

### 8.5 拜访历史

在账户/联系人/商机的详情页中，新增了"拜访记录" Tab，集中展示与该对象相关的所有拜访事件。

## 9. 销售区域

销售区域（Territory）是层级化的组织结构，用于管理销售团队的分工和客户分配。

### 9.1 区域结构

采用树状层级结构：

```
Global (全球)
├── Asia (亚太)
│   ├── Asia_RedRussia
│   ├── Asia_Libert
│   ├── Asia_Baji
│   ├── Asia_Taji
│   └── Asia_HHK
├── Europe (欧洲)
│   ├── Europe_EastTooth
│   └── Europe_PutiTooth
├── America (美洲)
│   └── America_NovaStar
├── Africa_GoldenSands
└── Oceania_DeepBlue
```

### 9.2 界面布局

销售区域页面采用左右分栏布局：

| 区域 | 内容 |
|:----:|------|
| **左侧（树）** | 区域层级树，支持搜索筛选、悬停显示操作按钮 |
| **右侧（详情）** | 选中区域后的详细信息面板 |

### 9.3 功能说明

**区域管理**（需要管理员/区域负责人权限）：

| 功能 | 操作 |
|------|------|
| 新建区域 | 点击树上方"新建"按钮 |
| 编辑区域 | 悬停区域名称 → 点击编辑图标 |
| 删除区域 | 悬停区域名称 → 点击删除图标 |

**区域详情面板**包含 5 个 Tab：

| Tab | 内容 |
|-----|------|
| **区域信息** | 名称、编码、类型、上级区域、描述 |
| **成员** | 该区域的成员列表（添加/移除） |
| **账户** | 关联到该区域的客户账户（关联/解除） |
| **产品** | 区域专属产品目录（含区域定价） |
| **管道** | 该区域下所有账户的商机管道汇总 |

## 10. 自定义对象

自定义对象允许管理员按需创建新的数据表，扩展系统功能。

> ⚠️ 此功能仅对管理员开放。

### 10.1 创建自定义对象

1. **入口**：侧边栏 → 管理 → 自定义对象
2. 点击"新建对象"
3. 填写基本信息：
   - 对象标签（显示名称）
   - 对象 API 名称
   - 对象说明
4. 添加字段：

| 字段类型 | 说明 |
|---------|------|
| 文本 | 单行文本，可设最大长度 |
| 多行文本 | 长文本内容 |
| 数字 | 整数或小数 |
| 货币 | 金额字段 |
| 日期 | 日期选择 |
| 日期时间 | 日期+时间 |
| 选择列表 | 下拉单选，自定义选项值 |
| 复选框 | 是/否 |
| 邮箱 | 邮箱格式 |
| 电话 | 电话号码 |
| 网址 | URL 格式 |
| 关联 | 关联到其他自定义对象 |

5. 点击"保存" → 系统自动创建对应的数据表

### 10.2 记录管理

创建自定义对象后，可以在对象管理页面中：
- **新建记录**：填写动态表单（根据定义的字段自动生成）
- **列表浏览**：表格展示所有记录
- **详情查看**：查看单条记录的完整信息
- **编辑/删除**：修改或删除记录

### 10.3 应用场景示例

| 对象 | 字段 | 用途 |
|------|------|------|
| 培训课程 | 名称、讲师、时长、状态、人数、开始日期 | 销售培训管理 |
| 项目 | 名称、预算、优先级、状态 | 内部项目跟踪 |

## 11. 工作流

工作流（Workflow）是条件-动作引擎，当满足特定条件时自动执行预设操作。

> ⚠️ 此功能仅对管理员开放。

### 11.1 工作原理

```
[触发事件] → [条件评估] → [动作执行]
    ↑              ↑              ↑
 创建/更新     多条件 AND    发送通知等
```

### 11.2 创建工���流规则

1. **入口**：侧边栏 → 管理 → 工作流
2. 点击"新建工作流"
3. 设置基本信息：
   - 规则名称
   - 对象类型（如：账户、商机）
   - 触发事件（创建时/更新时/创建或更新时）
4. 设置条件：
   - 选择字段、操作符（等于/大于/小于/包含/不为空等）、值
   - 多个条件之间是 AND 关系
5. 设置动作：
   - 选择动作类型（目前支持：系统通知）
   - 配置动作参数（通知消息内容）
6. 保存并启用

### 11.3 示例规则

| 规则名 | 条件 | 动作 |
|--------|------|------|
| 高金额商机通知 | 金额 ≥ 500,000 | 发送系统通知给管理员 |
| 赢单自动更新 | 阶段变为"赢单" | 发送系统通知给管理员 |

## 12. 报表与仪表盘

### 12.1 报表

报表用于数据分析和统计。

**创建报表**：
1. **入口**：侧边栏 → 管理 → 报表
2. 点击"新建报表"
3. 选择报表类型和对象
4. 配置字段、筛选条件、分组和聚合
5. 运行查看结果

**报表类型**：

| 类型 | 说明 |
|------|------|
| 表格 | 原始数据列表 |
| 汇总 | 按字段分组，支持聚合（求和/平均值/计数） |

### 12.2 仪表盘

仪表盘将多个报表组件组合到一个页面中，实现数据可视化。

**创建仪表盘**：
1. **入口**：侧边栏 → 管理 → 仪表盘
2. 点击"新建仪表盘"
3. 命名并保存
4. 添加组件：选择报表、设置标题、图表类型

**组件图表类型**：

| 类型 | 说明 |
|------|------|
| 表格 | 以表格形式展示数据 |
| 指标 | 以大数字突出显示汇总值 |

## 14. 常见问题

### 14.1 登录问题

**Q: 忘记密码怎么办？**
A: 请联系系统管理员重置密码。

**Q: 登录后跳转不到首页？**
A: 检查网络连接，确认后端服务是否正常运行。

### 14.2 数据操作

**Q: 为什么我创建的记录其他人看不到？**
A: 当前版本为单租户系统，所有用户共享数据。如果看不到，可能是筛选条件或搜索关键词导致的。

**Q: 删除的数据可以恢复吗？**
A: 目前不支持数据恢复，删除操作请谨慎执行。

### 14.3 拜访管理

**Q: 签到后忘记签退怎么办？**
A: 可以编辑拜访事件，手动填写结束时间和时长。

**Q: 可以关联多个账户到同一个拜访吗？**
A: 一个拜访事件只能关联一个 WhatId（账户/商机）。如需关联多个，可以创建多条拜访记录。

### 14.4 性能提示

**Q: 数据量很大时页面变慢怎么办？**
A: 善用搜索和筛选功能减少数据加载量。列表默认每页 20 条，可以调整分页大小。

**Q: 全局搜索支持哪些对象？**
A: 目前支持搜索账户名称。

### 14.5 技术支持

如有其他问题或功能建议，请联系系统管理员。
